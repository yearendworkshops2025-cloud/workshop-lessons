# L12: Build a Todo List App

## What is this lesson about?

Build a **real application** - a Todo List that saves your tasks permanently! Then publish it online for anyone to use.

## The Challenge

Ask Claude Code to:
1. **Create a Todo List application** with:
   - Add new tasks
   - Mark tasks as complete
   - Delete tasks
   - **Permanent storage** (tasks survive page refresh!)
2. **Make it look good** with CSS
3. **Publish it on Vercel** for the world to use!
4. **Create an HTML presentation** explaining how it works

## How to Start

```
cd ~/Lessons/L12 && claude+
```

Copy this and paste in terminal!

## What to Say to Claude
> "Create a Todo List web app with permanent storage. I should be able to add tasks, complete them, and delete them. Tasks should survive page refresh. Make it look nice and publish it on Vercel!"

## What You Will Learn

- How to build a complete web application
- How to store data permanently (localStorage or database)
- CRUD operations: Create, Read, Update, Delete
- How to deploy a real app

## Key Concepts

- **CRUD**: Create, Read, Update, Delete - the 4 basic data operations
- **Persistent Storage**: Data that stays even after closing the browser
- **localStorage**: Browser storage that persists
- **State Management**: Keeping track of your app's data

## Expected Features

- [ ] Input field to add new todos
- [ ] List of all todos
- [ ] Checkbox to mark complete
- [ ] Delete button for each todo
- [ ] Data persists on refresh
- [ ] Clean, usable design
